# Layout Rules v1 Draft

## 1. 目标

本文定义 v1 运行时的布局计算规则。

目标是：

- 接近网页常见布局逻辑
- 足够稳定可实现
- 足够适合 Cocos 运行时
- 足够支持 AI 生成后的自动排版

v1 重点支持：

- flex 布局
- box model
- absolute 定位
- 文本与图片测量接入

---

## 2. 基本术语

### 布局节点
每个 JSON 节点在运行时对应一个布局节点。

### 内容区
节点实际用于摆放子节点的区域。

内容区尺寸计算规则：

- `contentWidth = resolvedWidth - paddingLeft - paddingRight`
- `contentHeight = resolvedHeight - paddingTop - paddingBottom`

### 外部尺寸
节点最终占用的尺寸，包含自身内容和 padding，但不包含 margin。

### 主轴 / 交叉轴
- `flexDirection: row` 时，主轴为水平，交叉轴为垂直
- `flexDirection: column` 时，主轴为垂直，交叉轴为水平

---

## 3. 布局参与规则

### `display: none`
节点不参与：

- 测量
- 排版
- 渲染

### `position: absolute`
节点：

- 不参与普通 flex 流布局
- 但仍然是父节点的子节点
- 在普通子节点布局完成后单独计算位置

### 普通流节点
满足以下条件时参与 flex 流布局：

- `display !== none`
- `position !== absolute`

---

## 4. 尺寸解析规则

## 4.1 固定值
若 `width` 或 `height` 为数字，则直接作为解析后尺寸。

## 4.2 百分比
若 `width` 或 `height` 为百分比字符串：

- 相对父节点内容区尺寸计算
- 例如 `width: "50%"` 表示父内容区宽度的一半

说明：
- 若父内容区对应维度尚未确定，则该百分比尺寸在当前阶段视为待定值
- 待父节点尺寸确定后再次解析

## 4.3 `auto`
若为 `auto`：

- 容器节点由布局规则决定
- 文本节点由文本测量决定
- 图片节点由资源原始尺寸或另一维推导决定

## 4.4 最小最大约束
在节点得到候选尺寸后，应用：

- `minWidth / minHeight`
- `maxWidth / maxHeight`

优先级：

1. 原始候选尺寸
2. 应用最小约束
3. 应用最大约束

---

## 5. 盒模型规则

### `padding`
- 影响子节点布局区域
- 影响节点最终尺寸中的内容区大小

### `margin`
- 影响节点与同级节点之间的间距
- 不计入节点自身内容区

### `gap`
- 只作用于普通流子节点之间
- 不作用于 absolute 子节点
- 不需要最后一个流内子节点补 gap

### 间距叠加规则
v1 不实现网页中复杂的 margin collapse。

规则：

- 相邻节点间距 = 前一个节点 marginEnd + 后一个节点 marginStart + gap
- 上下或左右根据主轴方向取对应边值

---

## 6. Flex 容器布局流程

适用于：

- `type = view`
- `display = flex`

### 总流程

1. 解析容器自身尺寸
2. 扣除 padding，得到内容区
3. 收集普通流子节点
4. 测量每个子节点的基础尺寸
5. 计算主轴剩余空间
6. 处理 grow / shrink
7. 计算主轴位置分布
8. 计算交叉轴对齐
9. 写回子节点最终位置与尺寸
10. 最后处理 absolute 子节点

---

## 7. 子节点基础尺寸测量

### 7.1 普通容器子节点
若子节点是 `view`：

- 先根据显式 `width / height / flexBasis` 得到候选尺寸
- 若某维为 `auto`，则由子节点自身内容测量决定

### 7.2 文本子节点
文本节点测量规则：

- 若宽度固定，则以该宽度进行排版测量高度
- 若宽度为 `auto`，则先测单行自然宽度
- 若 `whiteSpace: normal` 且父布局对其提供了可用宽度约束，则允许换行
- 高度由文本排版结果给出

### 7.3 图片子节点
图片节点测量规则：

- 若宽高都显式给定，直接使用
- 若只有一维显式给定，另一维按原始比例推导
- 若都未给定，使用资源原始尺寸
- 若资源尚未加载完成，可先使用占位尺寸或延迟重排

---

## 8. 主轴尺寸分配

## 8.1 基础主轴尺寸
每个普通流子节点先得到一个基础主轴尺寸：

优先级建议：

1. 对应轴显式 `width / height`
2. `flexBasis`，若不是 `auto`
3. 内容测量尺寸

其中：

- `row` 使用宽度作为主轴尺寸
- `column` 使用高度作为主轴尺寸

## 8.2 剩余空间
容器主轴可用空间：

- 内容区主轴尺寸
- 减去所有普通流子节点基础主轴尺寸
- 减去所有 margin
- 减去所有 gap

得到剩余空间 `freeSpace`

## 8.3 `flexGrow`
若 `freeSpace > 0` 且存在 `flexGrow > 0` 的节点：

- 按 grow 权重分配剩余空间
- 子节点主轴尺寸 = 基础尺寸 + 分配增量

## 8.4 `flexShrink`
若 `freeSpace < 0`：

- 按 shrink 权重缩小子节点主轴尺寸
- 缩小后再应用 min/max 约束

说明：
- v1 中 shrink 算法可简化，不必完全复刻 CSS 的复杂公式
- 但必须保证总尺寸尽量落入容器内容区范围内

---

## 9. 主轴位置分布

在主轴总尺寸确定后，根据 `justifyContent` 决定子节点起始偏移与间隔分布。

支持：

- `flex-start`
- `center`
- `flex-end`
- `space-between`
- `space-around`
- `space-evenly`

说明：
- `gap` 先参与固定间距计算
- `justifyContent` 再分配剩余主轴空白

---

## 10. 交叉轴对齐

### 容器级：`alignItems`
决定子节点默认交叉轴对齐方式。

### 子节点级：`alignSelf`
若不是 `auto`，则覆盖 `alignItems`。

支持：

- `flex-start`
- `center`
- `flex-end`
- `stretch`

### `stretch`
仅在以下条件满足时生效：

- 子节点该交叉轴尺寸未显式指定
- 子节点不是 absolute

行为：
- 子节点交叉轴尺寸拉伸到容器内容区交叉轴尺寸减去对应 margin

---

## 11. 换行规则

v1 支持基础 `flexWrap: wrap`。

### 行分组规则
- 按子节点顺序依次放入当前行
- 若加入后超出当前主轴可用空间，则换到下一行
- 每行内部仍使用主轴布局规则

### 行间距
- 行与行之间使用 `gap`
- 行高或列宽取该行内最大交叉轴尺寸

说明：
- v1 不支持 `alignContent`
- 多行整体在交叉轴方向按从起点堆叠处理

---

## 12. Absolute 定位规则

## 12.1 包含块
`position: absolute` 子节点相对最近的父布局节点内容区定位。

v1 中简化为：

- 相对直接父节点内容区定位

## 12.2 定位优先级
支持属性：

- `left`
- `right`
- `top`
- `bottom`

规则：

- 若给了 `left`，优先确定水平起点
- 若同时给了 `left` 和 `right` 且宽度未指定，可用父内容区宽度扣除左右值得到宽度
- 垂直方向同理
- 若某方向未提供定位值，则默认使用该方向起点 `0`

## 12.3 absolute 尺寸
absolute 节点仍先按自身尺寸规则测量：

- 固定值
- 百分比
- auto
- 文本 / 图片内容测量

然后再根据定位属性修正位置或尺寸。

---

## 13. 文本测量规则

文本测量是布局系统的一部分，而不是纯渲染细节。

### 需要参与测量的因素
- `text`
- `fontSize`
- `lineHeight`
- 可用宽度
- `whiteSpace`

### 基本行为
- `nowrap`：按单行测量
- `normal`：允许在可用宽度下自动换行

### 与父布局的关系
- 在 `row` 布局中，文本常作为内容宽度来源
- 在 `column` 布局中，文本高度通常由换行结果决定

### 重排触发条件
以下变化应使文本节点及祖先布局失效：

- 文本内容变化
- 字号变化
- 行高变化
- 可用宽度变化

---

## 14. 图片测量规则

### 测量来源
- `props.src`
- 资源原始尺寸
- 显式 `width / height`
- `objectFit`

### 规则
- `fill`：按布局结果直接拉伸
- `contain`：保持比例，完整显示在目标框内
- `cover`：保持比例，覆盖目标框
- `none`：尽量使用原始尺寸

说明：
- `objectFit` 主要影响渲染显示和目标框内内容缩放
- 节点外部布局尺寸仍由 width/height/auto 规则决定

---

## 15. 布局计算顺序建议

建议采用自顶向下约束传递、自底向上尺寸回传的混合流程。

### 推荐步骤
1. 根节点获得外部可用尺寸
2. 自顶向下传递父内容区约束
3. 子节点根据约束进行测量
4. 自底向上汇总内容尺寸
5. 父节点完成自身布局分配
6. 写回所有节点最终 frame

---

## 16. 脏标记与重排

运行时需要脏标记系统。

### 需要触发布局失效的变化
- 子节点增删
- `style` 中布局相关属性变化
- 文本内容变化
- 图片资源尺寸变化
- 父节点尺寸变化

### 推荐脏标记类型
- `styleDirty`
- `measureDirty`
- `layoutDirty`
- `renderDirty`

### 重排范围
- 优先从变更节点向上冒泡到最近受影响祖先
- 避免整棵树全量重排

---

## 17. Cocos 映射建议

运行时布局结果至少应写回以下信息：

- 节点位置
- 节点尺寸
- 渲染透明度
- 层级顺序
- 文本内容与样式
- 图片资源与缩放模式

建议：

- 布局坐标系统先统一为左上角排版语义
- 最终再映射到 Cocos 节点坐标与锚点规则

---

## 18. v1 刻意不支持的内容

- 完整 CSS shrink/grow 细节兼容
- 完整 margin collapse
- 完整 baseline 对齐
- 完整 `alignContent`
- `inline` 排版
- 复杂文本富排版
- 复杂选择器和样式继承

这些能力可在 v2 以后再评估。
